**Usage:**
- Sign a JWT token when a user logs in.

### Async/Await Pattern

**Purpose:**
- **Consistency:** Ensures that asynchronous operations are handled consistently and predictably.

**Features:**
- **Readability:** Makes asynchronous code look and behave more like synchronous code.
- **Error Handling:** Easier to handle errors with try/catch blocks.

**Usage:**
- Use async functions for database operations to ensure they complete before proceeding.
- Use await to wait for the result of an asynchronous operation.

  const payload = { userId };
  const secret = process.env.JWT_SECRET; // Should be stored in environment variables

  const secret = process.env.JWT_SECRET;
  const decoded = jwt.verify(token, secret);
  return decoded;
}
