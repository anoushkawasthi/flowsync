**middleware/requireAuth.js**:

```javascript
const jwt = require('jsonwebtoken');

const requireAuth = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) {
    return res.status(401).send('Access Denied');
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (err) {
    res.status(400).send('Invalid Token');
  }
};

module.exports = requireAuth;
```

### High-Level Architecture

   - MongoDB database that stores user data, sessions, and revoked tokens.

### Flow of Data

1. **User Registration and Login**:
   - **Client**: Sends a registration request with user details.
   - **API**: Validates the request and stores the user in the database.
   - **Database**: Stores the user data in the `Users` collection.
   - **API**: Generates JWT access and refresh tokens.
   - **Client**: Receives the tokens and stores them for future requests.

2. **Authenticated Requests**:
   - **Client**: Sends a request with the JWT access token in the Authorization header.
   - **API**: Middleware verifies the token and extracts user information.
   - **Business Logic**: Processes the request and interacts with the database if needed.
   - **Database**: Retrieves or stores data as required.
   - **API**: Returns the response to the client.

3. **Token Refresh**:
   - **Client**: Sends a refresh token to the refresh endpoint.
   - **API**: Verifies the refresh token and generates new access and refresh tokens.
   - **Client**: Stores the new tokens and uses them for future requests.

4. **Token Revocation**:
   - **Client**: Sends a logout request.
   - **API**: Revokes the current access and refresh tokens.
   - **Database**: Stores the revoked tokens in the `TokenBlacklist` collection.

5. **Rate Limiting**:
   - **Client**: Sends a request to a rate-limited endpoint.
   - **API**: Middleware checks if the request is within the rate limit.
   - **Business Logic**: Processes the request if within the limit; returns an error if exceeded.

### Interaction Between Components

1. **Client and API Layer**:
   - The client sends HTTP requests to the API endpoints.

2. **API and Business Logic Layer**:

   - The business logic layer contains route handlers, middleware, and utility functions.

3. **Business Logic and Data Access Layer**:
   - The business logic layer interacts with the data access layer to perform database operations.
   - The data access layer contains models and database connection logic.
